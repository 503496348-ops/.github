# Security Policy

## 报告安全问题

如发现漏洞、密钥泄露、权限绕过、供应链风险或数据暴露，请不要公开发布利用细节。请通过仓库 Issue 使用安全报告模板提交最小复现信息，或联系项目维护者私下处理。

报告建议包含：

- 受影响仓库与版本/提交。
- 漏洞类型与影响范围。
- 复现步骤或证据截图。
- 是否涉及真实密钥、用户数据、客户信息。
- 建议修复方向。

## 处理优先级

- P0：密钥泄露、远程代码执行、任意数据读取/写入、生产凭证暴露。
- P1：认证/授权绕过、供应链高危漏洞、敏感信息泄露。
- P2：中低危依赖漏洞、错误配置、日志泄露。
- P3：安全文档或流程改进。

## 维护要求

- 禁止提交 `.env`、私钥、token、真实客户数据。
- 新增依赖前检查许可证与安全风险。
- 对外文档必须脱敏真实 IP、服务器名、内部路径和架构细节。
