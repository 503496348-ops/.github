# Public Roadmap

本路线图服务于 503496348-ops 公共仓库治理与 Skill 项目化。

## 现在：Community Health Baseline

目标：所有公开仓库至少有默认治理兜底。

- [x] 账号级 `.github` 治理仓。
- [x] Issue 模板、PR 模板、安全政策、贡献说明。
- [x] 默认组织 Profile。
- [x] Funding 入口。
- [ ] 活跃产品仓 Topics/Description 标准化。
- [ ] 活跃产品仓 README Quickstart 审计。

## 下一步：Skill-to-Project Incubation

目标：筛出可公开、可运行、可赞助的 Skill 项目候选。

- [ ] 建立 Skill 项目化候选清单。
- [ ] 每个候选按 G1-G6 打分。
- [ ] 选出第一批 L1 Incubating 项目。
- [ ] 为 L1 项目补齐 README、LICENSE、示例、smoke test。
- [ ] 将不可公开 Skill 标记为 Internal-only，避免误发布。

## 再下一步：Quality Gate Automation

目标：从人工审计升级为自动化治理。

- [ ] 仓库健康扫描脚本：README/LICENSE/SECURITY/CI/Topics。
- [ ] 敏感词与外部污染扫描脚本。
- [ ] Sponsor-ready 检查器。
- [ ] 产品矩阵与 GitHub 仓库数据一致性校验。
- [ ] 定期报告：只发一份完整日报，避免碎片化刷屏。

## 长期：Open Product Studio

目标：形成可持续的公开产品矩阵。

- [ ] 每个成熟项目有明确 owner、路线图和版本节奏。
- [ ] 建立 Good First Issues 池。
- [ ] 建立贡献者指南与示例贡献路径。
- [ ] 对高价值项目开放 GitHub Sponsors 叙事和支持边界。
- [ ] 建立项目退役机制：长期不维护项目归档，避免矩阵膨胀。

## 反目标

- 不把所有 Skill 都强行公开。
- 不为了数量制造空壳仓库。
- 不用 README 伪装产品。
- 不隐藏外部来源和许可证义务。
- 不承诺无法兑现的支持 SLA。
