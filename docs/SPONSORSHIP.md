# Sponsorship Policy

503496348-ops 的赞助入口用于支持公开 AI 工具、Skill 项目化、文档维护、测试样例、社区治理和长期安全更新。

Sponsor button: https://github.com/sponsors/503496348-ops

## 赞助资金优先用途

1. **维护公共工具**：修复 bug、升级依赖、补齐文档和示例。
2. **项目化内部 Skill**：把可公开的自动化能力打磨成独立仓库。
3. **质量门禁**：补 CI、测试样例、演示数据、回归脚本。
4. **安全治理**：漏洞响应、依赖扫描、敏感信息清理。
5. **社区体验**：Issue triage、good first issue、贡献者引导。

## 赞助不购买什么

- 不购买私有客户数据访问权。
- 不购买跳过安全审查或代码审查的权利。
- 不购买违反许可证的代码整合。
- 不购买未公开承诺的专属 SLA。

## 可赞助项目的最低标准

进入 Sponsored-ready 状态的仓库必须至少具备：

- 可运行 Quickstart。
- 至少一个可验证示例。
- LICENSE、SECURITY、SUPPORT、CONTRIBUTING。
- Roadmap 和公开 Issue。
- 敏感信息与外部污染清理记录。

## 赞助层级建议

| 层级 | 适合对象 | 回报边界 |
|---|---|---|
| Supporter | 使用者、关注者 | 支持维护，公开致谢可选 |
| Builder | 依赖项目的团队 | 优先关注 bug 报告和文档缺口 |
| Partner | 深度使用方 | 可参与路线图讨论，不承诺绕过治理门禁 |

具体金额和平台能力以 GitHub Sponsors 页面为准。

## 透明度

每个 Sponsored-ready 项目应在 README 或 Roadmap 中说明：

- 当前维护状态。
- 近期重点。
- 需要赞助支持的具体事项。
- 赞助无法覆盖的事项。
